1. OVERVIEW
-----------

"My Child: A Profile" is a 22-section guided profiling tool that helps parents
of neurodivergent children document who their child is — strengths, needs,
experiences, and context — in their own words. It generates a professionally
formatted PDF report combining the parent's answers with AI-generated insights
written in Rich Ferriman's voice.

The tool is entirely client-side for PDF generation (jsPDF). AI processing
uses Claude Sonnet 4 via an edge function. Progress can be saved and restored
using an 8-character access code (stored encrypted, 14-day expiry).

URL: /my-child-profile
Navigation: Featured in Parent Guides menu and on the Start page.

4. QUESTION TYPES
------------------

- "text": Single-line free text (renders as 3-row textarea)
- "text-large": Multi-line free text (renders as 6-row textarea)
- "single-select": Radio-style selection from 3-6 predefined options (rendered as clickable buttons)

Child voice questions are always free text (no type field, rendered as 3-row textareas).
They are stored with "cv_" prefix on the question ID.


5. DATA MODEL
--------------

Context: ChildProfileContext.tsx

ChildProfileState {
  setup: {
    childName: string
    filledBy: string
    sharedWith: string[]
    reason: string
  }
  sections: Record<number, {
    answers: Record<string, string | string[]>  // keyed by question ID
    reflection: string
  }>
  finalStatement: string
}

State is held in React context (useState). Not persisted to localStorage.
State is lost on page refresh unless saved via access code.

Section status logic:
  - "empty": no answers and no reflection
  - "complete": any answer or reflection present
  - Note: there is no true "in-progress" state — any content = complete


6. SAVE & RESTORE SYSTEM
--------------------------

Edge function: supabase/functions/save-profile/index.ts
Database table: saved_profiles

Schema:
  - id: UUID (primary key)
  - access_code: string (8 chars, uppercase alphanumeric excluding I/O/0/1)
  - profile_data: JSONB (full ChildProfileState)
  - stage: string ("opening" | "builder")
  - active_section: integer
  - created_at: timestamptz
  - expires_at: timestamptz (default: 14 days from creation)

Behaviour:
  - Save: generates new 8-char code or updates existing if code provided
  - Load: retrieves profile by access code if not expired
  - Cleanup: calls cleanup_expired_profiles RPC on each request
  - Collision avoidance: retries code generation up to 10 times


7. AI REPORT GENERATION
-------------------------

Edge function: supabase/functions/generate-profile-report/index.ts

AI Model: Claude Sonnet 4 (claude-sonnet-4-20250514)
API: Anthropic direct (requires ANTHROPIC_API_KEY secret)
Max tokens: 4000
Temperature: 0.7

Input: buildProfileText() in FinalScreen.tsx assembles all answered questions
with their labels into a structured text document, organised by section.
Format: "Question label\nAnswer value" pairs grouped under section headings.

Knowledge base augmentation: Before sending to the AI, the edge function searches
the knowledge_chunks table using full-text search (top 5 results) and prepends
relevant passages as context for the AI to draw upon.

System prompt rules:
  - UK English only (behaviour, recognise, mum)
  - No em dashes, no exclamation marks, no corporate language
  - No Americanisms (great job, awesome, reach out, etc.)
  - Voice: direct, grounded, human — speaks to parents
  - CRITICAL: Child is never described with deficit/damage language
  - Banned words when applied to child: broken, damaged, failing, lost, etc.
  - Replacement framing: "not yet supported", "let down by the system", etc.

Output structure per section (max 200 words each):
  Paragraph 1: Reflect — mirrors what the parent described
  Paragraph 2: Context — explains why this pattern exists
  Paragraph 3: Practical strategies — specific to this child

Special blocks generated:
  1. Opening line: "This profile was built by someone who knows this child
     better than any system ever will. Read it as such."
  2. Ways of Working: 5-7 prioritised strategies in paragraph form
  3. Some Things That May Help: Max 5 "Have you tried..." suggestions
  4. Conclusion: One-page standalone summary (max 400 words)

12. KNOWN ISSUES AND GAPS
---------------------------

1. TEST DATA BUTTON IN PRODUCTION
   The "Load test data (all sections filled)" button is visible on the
   opening screen in production. It should be hidden behind an environment
   check or admin flag.

2. NO BEFOREUNLOAD WARNING
   Users can accidentally navigate away or close the tab and lose all
   progress. No window.onbeforeunload handler is implemented.

3. ANONYMISED SUMMARY SHARING NOT WIRED
   The post-generation consent screen offers "Share anonymised summary"
   but the button only updates local UI state. No data is actually sent
   to the backend.

4. NO "IN-PROGRESS" STATUS
   Section status jumps from "empty" to "complete" with any content.
   There is no partial-completion status. The "in-progress" (pencil)
   icon code exists in ProfileSidebar but is never triggered.

5. MAX TOKENS MAY BE TIGHT
   With the addition of the Conclusion block, the 4000 max_tokens limit
   may risk truncation on profiles with many completed sections.
   Consider increasing to 5000.

6. STATE NOT IN LOCALSTORAGE
   Profile state is only in React context. A page refresh loses everything
   unless the user has manually saved with an access code.

7. NO PRINT/EXPORT OF INDIVIDUAL SECTIONS
   Users cannot export or print a single section — only the full report.

8. NO PROGRESS INDICATOR IN BUILDER
   Beyond the sidebar status icons, there is no overall completion
   percentage or progress bar.
