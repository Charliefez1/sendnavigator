

## Plan: Contextual Follow-ups and Reading Depth for Ask Rich

### What this adds

Two enhancements to make Ask Rich answers richer and more personally relevant:

**1. Reading depth toggle on answers**

After receiving an answer, the user sees a compact "key points" summary by default, with a "Read more detail" expandable section containing the full structured answer. Uses the existing `ReadingDepthSection` component pattern already in the codebase.

- The `parentGuide` tab shows: key points (title + first overview paragraph + encouragement) collapsed, full guide (all overview, whatHelps, whatToAvoid) expanded on click
- The policy tab shows: plain answer as key points, full structured sections (whatWeKnow, whatWeDoNotKnow, clarifications) as more detail
- No tabs needed when parentGuide is absent — just the reading depth toggle on the single answer

**2. Contextual follow-up questions after an answer**

After an answer is displayed, show 2–3 contextual follow-up prompts based on the topic. These are generated by the AI as part of the response.

Changes to the edge function: add a `followUps` field to the JSON response format:
```json
{
  "followUps": [
    "What does PDA look like in a 7-year-old?",
    "How do I explain PDA to school?",
    "What helps with demand avoidance at home?"
  ]
}
```

The system prompt instructs the AI to generate 2–3 follow-up questions that would naturally help a parent go deeper — including age-specific, gender-specific, or behaviour-specific angles when relevant.

On the frontend, these render as clickable chips below the answer. Clicking one submits it as a new question automatically.

### Changes required

**Edge function (`supabase/functions/qanda/index.ts`)**
- Add `followUps` to the JSON response format in the system prompt (array of 2–3 strings)
- Instruct the AI: "Generate 2–3 natural follow-up questions a parent might want to ask next. Make these specific and contextual — if the topic involves a condition, suggest age-specific, school-specific, or home-strategy angles."

**Types (`src/components/qanda/types.ts`)**
- Add `followUps?: string[]` to `QandAAnswer`

**Answer display (`src/components/qanda/AnswerDisplay.tsx`)**
- Wrap the parent guide content in reading depth sections (key points / full detail)
- Wrap the policy content in reading depth sections
- Render follow-up question chips below the answer

**Q&A component (`src/components/qanda/QandAComponent.tsx`)**
- Pass an `onFollowUp` callback to `AnswerDisplay` that triggers a new question submission when a follow-up chip is clicked

### What this achieves

A parent asking "Tell me about PDA" sees:
- **Key points** visible immediately (digestible, not overwhelming)
- **"Read more detail"** expandable for the full depth
- **Follow-up chips**: "What does PDA look like at secondary school?" / "How do I talk to school about PDA?" / "What strategies help at home?"

Clicking a follow-up sends a new, more specific question — naturally guiding parents deeper without them having to figure out what to ask next.
